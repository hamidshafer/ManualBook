11.3Tasks

Tasks are utilised for the following reasons:

â€¢An activity is required some time in the future

In any of the above cases it is useful to create a task, which is either linked to a particular client or independent of a client account and meant to be executed not in the context of a client account.

11.3.1Create a task

If a task is required for a specific account it is best practise to first select the client account within CLIENT MANAGEMENT (see chapter 7).

Open the Pull Down Context Menu (left click mouse), move down to NEW TASK and activate the dialogue through a left click of the mouse. You may do this from anywhere within the Client Tab Menu. It is not required to select

TASKS first. The new task dialogue opens and you can fill in the required fields.

In case you wish to use a predefined task definition, click on â€œSelect Task

Definition...â€? (see 08) and select the appropriate entry. A left mouse click will load the task definition and will also pre-populate the Title field (see 01).

The task definition will also contain a specified time period for the task and a due date will autimatically be calculated. In case the due date needs to be set manually a left mouse click on the Due date field (see 02) will activate a date finder pop up. The current date is marked with a circle for easy

reference. Select a date and when you click on a day cell the pop up will close and the date is copied into the field. You can find a detailed description of the date pop up in chapter 15.

You may assign a member of your team to the task using field Assigned To (see 03). The client field is pre-populated with the client name as you created the task in the context of the client account (see 04).

The Description field is also pre-filled in case you use a predefined task definition. Otherwise you can write text into the field manually (see 05).

Labels can be added to each task (see 06). If creted from a pre-defined task labels will be copied from the task definition. Additional labels can be added.

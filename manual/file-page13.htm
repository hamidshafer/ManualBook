11.3.2Benefit of task definitions

Pre-defined task definitions can contain Instructions, Checklists and

Communication Templates. This increases productivity as well as compliance to a large degree.

Labels can be predefined and increase conformity of time lines and better identification of situations requiring special attention. See chapter 3 and 15 for more information about labels.

Instructions will assist the user in understanding and executing the task better. Every task definition also contains a specified time period, which is typical to execute the task. This helps staff to stay on top of their duties and the system can greatly assist to track due and overdue tasks.

A Checklist defines the deliverables of a task and each position is ticked off when completed.

Using a pre-defined task definition greatly increases efficiency and reduces potential for errors.

11.3.3Using a Communication Template

Once the task has been activated and was created from a pre-defined definition, Instructions, a Checklist and Communication Templates are displayed on the dialogue page if they have been added to the definition previously.

To activate a communication template simply click on its title and the template will open. The template can be filled and sent as described in chapter 8.

11.3.4Updating a task

If the task contains a Checklist with deliverables for the task each position can be ticked off and the result can be saved. Several communication templates may be associated with the task and each may apply to a position in the checklist. The system does not keep track of this relationship. It is up to

the user to decide which template to use under which circumstances. The instructions should clarify this.

Once all positions of the Checklist are ticked off the Status of the task can be set to DONE and the content needs to be saved. The system will update the status in the timeline and all other references where the task has been linked.
